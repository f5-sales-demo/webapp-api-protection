يشرح هذا الدليل بناء **مسار تسليم آمن لتطبيقات الويب** على F5 Distributed
Cloud، مُدار بواسطة Terraform. تغطي التكرارة 1 الأساس؛ وتُضاف مراحل الأمان مع نضوج
الخطة. كل مرحلة عديمة التأثير الجانبي (idempotent): إعادة تشغيلها تتقارب إلى الحالة نفسها.

## مراحل التمرين

| المرحلة | الهدف | الحالة |
| --- | --- | --- |
| البناء | نشر موازِن تحميل HTTP + تجمّع مصادر (httpbin) باستخدام Terraform؛ وربط الحالة البعيدة والتكامل المستمر | التكرارة 1 |
| التحقق | التأكد من أن موازِن التحميل يقدّم httpbin وأن الخطة عديمة التأثير الجانبي | التكرارة 1 |
| الحماية — WAF | إرفاق جدار حماية لتطبيقات الويب (`xcsh_app_firewall`) | مُخطّط له |
| الحماية — API | إضافة حماية API (`xcsh_api_definition`، اكتشاف API) | مُخطّط له |
| التفكيك | تدمير الكائنات المُدارة بعد تأكيد صريح | التكرارة 1 |

## المتطلبات المسبقة

- **Terraform** ≥ 1.5.
- الموفّر **`f5-sales-demo/xcsh`** ≥ 3.63.0، مُحلّل من سجل Terraform — أو بناء
  محلي عبر `dev_overrides`.
- رمز **API لـ F5 XC** لمستأجر التجهيز، و**وصول VPN** للوصول إليه.
- **Azure CLI** مُسجّل الدخول بصلاحية Contributor على الاشتراك (خلفية الحالة مشتركة مع
  حالة استخدام DNS وجرى تمهيدها بالفعل).

<Aside type="caution">
تستهدف التكرارة 1 مستأجر التجهيز (غير الإنتاجي). لا ترقِّ إلى ما قبل الإنتاج إلا بعد أن تكون
الخطة خالية من العيوب مقابل التجهيز.
</Aside>
